PaymentManager: 収支情報管理サービス
====================================

本書では収支情報管理サービスの仕様を記載する

サービス全体構成
----------------

.. uml::

   left to right direction
   skinparam packageStyle rect
   actor 利用者
   entity データベース

   rectangle Adhafera {
     boundary 登録画面
     boundary 検索画面
     control 登録コントローラー
     control 検索コントローラー
     利用者 -- 登録画面
     利用者 -- 検索画面
     登録画面 -- 登録コントローラー
     検索画面 -- 検索コントローラー
   }

   boundary Webブラウザ
   利用者 -- Webブラウザ

   rectangle Algieba {
     登録コントローラー -- (収支を登録する)
     登録コントローラー -- (収支を計算する)
     検索コントローラー -- (収支を検索する)
     Webブラウザ -- (収支を登録する)
     Webブラウザ -- (収支を検索する)
     Webブラウザ -- (収支を削除する)
     Webブラウザ -- (統計情報を確認する)
     Webブラウザ -- (辞書を登録する)
     (収支を登録する) -- データベース
     (収支を検索する) -- データベース
     (収支を削除する) -- データベース
     (収支を計算する) -- データベース
     (統計情報を確認する) -- データベース
     (辞書を登録する) -- データベース
   }

- 利用者は Android アプリ(Adhafera)、Webブラウザを使って収支を管理する
- 収支情報は全てサーバー内のデータベースで一括管理される
